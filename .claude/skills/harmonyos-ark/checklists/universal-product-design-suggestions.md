# 通用建议：功能丰富度 + 深色模式 + 多端适配

## 1. 功能点丰富度（通用）
1. 至少 3 条完整业务链路
- 每条链路都包含入口、处理过程、结果反馈。
2. 每条链路都包含异常分支
- 弱网、拒权、空数据、服务异常至少覆盖 2 种。
3. 每条链路都有可验证结果
- 用户可看到明确成功或失败状态。

## 2. 深色模式（通用）
1. 支持系统主题跟随
- 不建议仅保留手动开关而忽略系统主题。
2. 保证对比度
- 文本、图标、按钮在深色背景下可辨识。
3. 重点页面必测
- 首页、详情页、提交流程页、设置页。

## 3. 多端适配（手机/平板/PC）
1. 断点布局
- 手机: < 600dp
- 平板: 600dp - 840dp
- PC: >= 840dp
2. 交互一致
- 核心操作路径保持一致，不因设备变化而缺失功能。
3. 信息密度分级
- 手机优先任务流，平板/PC增加并行信息与效率操作。

## 4. 提审前最小验收
1. 三端都可完成核心业务链路
2. 深色模式下关键页面可读可点
3. 素材、文案、功能一致
4. 审核账号可复现主要流程

## 5. 适用场景
- 新项目立项时定义产品基线
- 版本提审前做快速体检
- 团队评审时统一口径
