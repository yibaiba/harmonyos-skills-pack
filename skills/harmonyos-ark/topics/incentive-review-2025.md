# 2025 创作激励与审核避坑主题

## Scope
- 围绕 2025 年鸿蒙开发者创作激励活动规则，提供应用设计与上架审核避坑指导
- 目标是减少因规则不匹配、材料缺失、内容违规导致的卡审

## Key Concepts
- 创作激励规则

- 内容合规
- 上架审核
- 素材与版权
- 隐私与权限说明
- 资质材料完整性

## 风险分级执行（提审前）
1. High
- 包含支付、账号体系、个人敏感信息、未成年人内容、UGC 内容。
- 要求：必须完成合规材料复核与测试回归后再提审。
2. Medium
- 包含定位、推送、媒体上传、第三方 SDK、后台任务。
- 要求：完成权限说明、异常流程验证、弱网与拒权测试。
3. Low
- 工具类、离线类、本地能力为主且无敏感权限。
- 要求：完成基础稳定性、文案真实性与素材版权检查。

## 弹窗频率与合规控制

华为审核对"频繁弹窗"零容忍，以下为硬性要求：

| 规则 | 说明 |
|------|------|
| **同类弹窗 ≤ 1 次/启动** | 同一功能（如评分、更新、广告）每次冷启动最多弹出一次 |
| **弹窗间隔 ≥ 30 s** | 连续不同弹窗之间至少间隔 30 秒 |
| **必须可关闭** | 所有弹窗必须有明确的关闭/跳过按钮，不可仅提供"同意" |
| **无强制阻断** | 禁止用弹窗阻断核心功能（如不同意协议就无法使用基础功能） |
| **开屏广告 ≤ 5 s** | 带"跳过"按钮，倒计时结束自动关闭 |

**豁免场景**（不计入弹窗频率）：
- 系统权限弹窗（由系统触发，非应用控制）
- 首次使用引导（仅首次安装后展示）
- 安全验证弹窗（支付确认、二次验证）

**常见拒审案例**：
1. 启动后连续弹出"通知授权 → 评分 → 广告"三个弹窗
2. 关闭弹窗后页面操作立即触发新弹窗
3. 弹窗无关闭按钮，仅有"去开启/去评分"

## Quick Q&A
- 按 2025 激励规则，应用题材和能力接入应优先满足哪些要求
- 上架审核最容易卡住的材料有哪些，如何一次准备齐全
- 涉及账号登录、支付、用户数据时，审核重点会卡在哪些项
- 宣传素材和截图怎么准备才能降低驳回风险

## 审核避坑清单（可执行）
1. 功能真实性
- 描述、截图、视频与应用实际功能保持一致，避免“文案承诺大于功能实现”。
2. 权限最小化
- 仅申请必要权限，并在应用内提供清晰用途说明和拒绝后的降级路径。
3. 隐私与数据合规
- 提供可访问的隐私政策，说明数据采集、用途、存储周期与注销机制。
4. 版权与素材合规
- 图标、截图、视频、字体、音乐均需具备合法授权，避免侵权素材。
5. 账号与资质一致性
- 开发者主体、应用信息、备案与资质材料保持一致，避免主体不一致导致驳回。
6. 版本可审性
- 提交版本需可稳定运行，避免崩溃、空白页、核心链路不可用。
7. 激励规则匹配
- 作品类型、技术栈、交付形态和活动要求逐条对照，避免因不满足活动条件失去激励资格。

## Common Pitfalls
- 只关注功能开发，忽略审核材料完整性
- 活动规则理解停留在旧版本，未核对当期口径
- 权限说明写得过于笼统，无法支撑审核判断
- 宣传素材与应用内容不一致，触发真实性问题

## Boundaries
- 不替代官方审核结论，最终以官方审核结果为准
- 不提供法律意见，仅提供合规准备建议与文档路由
- 若规则版本冲突，优先以当期官方活动页为准
