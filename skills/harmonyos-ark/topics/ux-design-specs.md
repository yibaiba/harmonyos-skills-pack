## UX 验收总览

**验收维度速览**

| 维度 | 核心关注点 | 严重度 |
|------|-----------|--------|
| 视觉风格 | 色彩/字体/图标符合 HDS 规范 | P1 |
| 布局 | 响应式、无裁切、安全区适配 | P0 |
| 动效 | 转场/手势/属性符合物理引擎规范 | P1 |
| 系统特性 | 导航条/状态栏/深色模式/启动页 | P0 |
| 多设备 | 折叠屏/平板/横竖屏/多窗口 | P0 |
| 人机交互 | 手势/光标/焦点/键盘 | P1 |

**快速决策树**

```
页面是否适配深色模式？─── 否 ──→ ❌ P0 必修
  │ 是
  ▼
布局是否用相对单位（vp/%）？── 否 ──→ ❌ P0 必修
  │ 是
  ▼
导航条 28vp 避让是否到位？── 否 ──→ ❌ P0 必修
  │ 是
  ▼
挖孔区内容是否被遮挡？── 是 ──→ ❌ P0 必修
  │ 否
  ▼
进入各维度逐项检查 ──→ 📋 验收清单
```

## 🎨 视觉风格

### 2.1 色彩

> 来源: https://developer.huawei.com/consumer/cn/doc/design-guides/color-0000001776857164

| 验收项 | 通过标准 | 判定 |
|--------|---------|------|
| 系统底色 | 使用宇宙蓝 + 雪域灰作为基础色调 | ✅ 符合 / ❌ 自定义底色未适配 |
| 品牌色突出度 | 品牌色仅用于强调元素，不喧宾夺主 | ✅ 适度 / ⚠️ 面积过大 |
| 色彩分层 | 前景与背景有明确对比层次 | ✅ 层次清晰 / ❌ 层次混乱 |
| 对比度 | 可读内容对比度 ≥ 3:1 | ✅ ≥ 3:1 / ❌ < 3:1 |
| 深色模式映射 | 色彩在深色模式下正确映射变化 | ✅ 已映射 / ❌ 未适配 |
| 色彩可辨识 | 相邻色 △Euv ≥ 20，色觉障碍可区分 | ✅ ≥ 20 / ⚠️ 接近临界 / ❌ < 20 |

**ArkUI 提示**: 使用 `$r('sys.color.ohos_id_color_*')` 系统资源色，深色模式自动映射。

### 2.2 字体

> 来源: https://developer.huawei.com/consumer/cn/doc/design-guides/font-0000001828772001

| 验收项 | 通过标准 | 判定 |
|--------|---------|------|
| 默认字体 | 使用 HarmonyOS Sans（无衬线）| ✅ 使用 / ❌ 未使用系统字体 |
| 信息层级 | 标题/正文/辅助文字大小层级清晰 | ✅ 清晰 / ⚠️ 层级不明显 |
| 最小可读字号 | 正文 ≥ 12fp，辅助文字 ≥ 10fp | ✅ 符合 / ❌ 字号过小 |
| 大字体支持 | 响应系统辅助功能大字体设置 | ✅ 支持 / ❌ 固定字号不响应 |
| 字体风格一致 | 全应用字体风格统一，无混用 | ✅ 统一 / ⚠️ 个别不一致 |

**ArkUI 提示**: 使用 `fp` 单位（跟随系统字体大小），避免 `px` 固定值。

### 2.3 图标

> 来源: https://developer.huawei.com/consumer/cn/doc/design-guides/application-icon-0000001953444009

| 验收项 | 通过标准 | 判定 |
|--------|---------|------|
| 图标风格 | 轻拟物风格，半透明模糊质感，分层设计 | ✅ 符合 / ❌ 扁平/拟物过重 |
| 网格对齐 | 图标元素在标准网格内视觉平衡 | ✅ 对齐 / ⚠️ 略偏 |
| 色彩渐变 | 上亮下暗渐变，色彩过渡柔和 | ✅ 符合 / ⚠️ 过渡生硬 |
| 多设备适配 | 手机/折叠屏/平板/2in1 四端图标清晰 | ✅ 四端适配 / ❌ 缺少适配 |

## 📐 布局

### 3.1 基础布局

> 来源: https://developer.huawei.com/consumer/cn/doc/design-guides/design-layout-basics-0000001795579413

| 验收项 | 通过标准 | 判定 |
|--------|---------|------|
| 禁止固定像素 | 使用响应式/弹性布局，不使用 `px` 固定值 | ✅ 响应式 / ❌ 固定像素 |
| 移动优先 | 从最小屏幕开始设计，向大屏渐进增强 | ✅ 遵循 / ⚠️ 未明确 |
| 布局完整性 | 折叠/旋转后无裁切、变形、错位 | ✅ 完整 / ❌ 有裁切或错位 |
| 安全区适配 | 内容避开挖孔区/刘海区/圆角区 | ✅ 已避让 / ❌ 内容被遮挡 |
| 断点系统 | 使用 600vp / 840vp 断点适配设备 | ✅ 使用 / ⚠️ 未使用断点 |
| 栅格系统 | 使用 `GridRow` / `GridCol` 栅格布局 | ✅ 使用 / ⚠️ 手动计算 |

**ArkUI 提示**: 使用 `.expandSafeArea()` 实现安全区扩展；用 `GridRow({ breakpoints: { value: ['600vp', '840vp'] } })` 定义断点。

### 3.2 响应式架构

> 来源: https://developer.huawei.com/consumer/cn/doc/design-guides/design-responsive-layout-structure-0000001748539684

| 场景 | 触发条件 | 布局变化 |
|------|---------|---------|
| 侧边 Tab | 宽度 ≥ 840vp | 底部 Tab → 侧边 Tab |
| 分栏视图 | 宽度 ≥ 600vp | 父子页面 → 左右分栏（默认 4:6） |
| 侧边导航 | 宽度 ≥ 840vp | 顶部导航 → 侧边导航栏 |

### 3.3 响应式布局方法

> 来源: https://developer.huawei.com/consumer/cn/doc/design-guides/design-responsive-layout-method-0000001795698449

**8 种布局方法**：拉伸、缩放、隐藏、折行、均分、延伸、挪移、重排

| 验收项 | 通过标准 | 判定 |
|--------|---------|------|
| 相对定位 | 使用百分比/权重，不用绝对坐标 | ✅ 相对 / ❌ 绝对定位 |
| 布局方法选择 | 根据内容特性选择合适的响应式方法 | ✅ 合理 / ⚠️ 单一方法 |
| 内容优先级 | 屏幕缩小时优先隐藏次要内容 | ✅ 分级处理 / ❌ 平铺截断 |

## ✨ 动效

### 4.1 转场动效

> 来源: https://developer.huawei.com/consumer/cn/doc/design-guides/transition-animation-0000001750078488

| 验收项 | 通过标准 | 判定 |
|--------|---------|------|
| 层级关系 | 同级/父子/跨级转场动效正确匹配 | ✅ 匹配 / ❌ 层级混乱 |
| 动效编排 | 分解场景 → 分类元素 → 协调运动 | ✅ 编排有序 / ⚠️ 缺少编排 |
| 一镜到底 | 复杂转场使用连贯过渡，无跳切 | ✅ 连贯 / ⚠️ 有跳切感 |

**ArkUI 提示**: 使用 `pageTransition` / `sharedTransition` 实现一镜到底。

### 4.2 手势动效

> 来源: https://developer.huawei.com/consumer/cn/doc/design-guides/gesture-animation-0000001797038213

| 验收项 | 通过标准 | 判定 |
|--------|---------|------|
| 触摸反馈 | 按下瞬间有视觉反馈 | ✅ 即时 / ❌ 无反馈或延迟 |
| 滑动跟手 | 滑动过程中内容跟随手指移动 | ✅ 跟手 / ❌ 卡顿或不跟手 |
| 离手减速 | 松手后继承速度并自然减速 | ✅ 自然 / ❌ 突然停止 |
| 翻页状态 | 翻页有位置限制和成功/失败态 | ✅ 有状态 / ⚠️ 缺失 |
| 捏合缩放 | 跟随自然手势物理特性 | ✅ 自然 / ❌ 非线性异常 |
| 物理引擎 | 使用摩擦/弹性/碰撞物理模型 | ✅ 使用 / ⚠️ 简单线性 |

**ArkUI 提示**: 使用 `springMotion()` / `responsiveSpringMotion()` 做物理动效。

### 4.3 动效属性

> 来源: https://developer.huawei.com/consumer/cn/doc/design-guides/animation-attributes-0000001797117229

| 属性 | 标准值 | 判定 |
|------|-------|------|
| 简单动效时长 | 100ms | ✅ ≈100ms / ❌ 偏差过大 |
| 局部动效时长 | 200ms | ✅ ≈200ms / ⚠️ 略偏 |
| 全屏动效时长 | 350ms | ✅ ≈350ms / ⚠️ 略偏 |
| 缓动曲线 | 必须使用缓动曲线，禁止线性 | ✅ 缓动 / ❌ 线性运动 |
| 帧率 | ≥ 60fps（旗舰机推荐 120fps） | ✅ ≥60fps / ❌ <60fps |

**ArkUI 提示**: 使用 `Curve.EaseInOut` / `Curve.Friction` 等预置曲线，禁用 `Curve.Linear`。

### 5.2 状态栏

### 5.3 深色模式

> 来源: https://developer.huawei.com/consumer/cn/doc/design-guides/dark-mode-0000001823255497

| 验收项 | 通过标准 | 判定 |
|--------|---------|------|
| 正文对比度 | 正文文字对比度 ≥ 5:1，最高 ≤ 17.6:1 | ✅ 在范围内 / ❌ 超出 |
| 控件对比度 | 控件与页面背景对比度 ≥ 2.2:1 | ✅ ≥ 2.2:1 / ❌ 不足 |
| 色彩辨识 | △Euv ≥ 20 | ✅ ≥ 20 / ❌ 不可区分 |
| 深色切换 | 必须切换为深色背景，不可保持浅色 | ✅ 已切换 / ❌ 仍为浅色 |
| 无黑白跳变 | 页面间无突兀的黑白切换 | ✅ 过渡自然 / ❌ 有跳变 |
| 无大面积白 | 无大面积白色图片或背景 | ✅ 无 / ❌ 有刺眼白块 |
| 语义色一致 | 警告=红色、拨号=绿色等语义不变 | ✅ 一致 / ❌ 语义色混乱 |
| 控件风格统一 | 同类控件深色模式下风格一致 | ✅ 统一 / ⚠️ 个别不一致 |
| 层级亮度 | 通过亮度差表达卡片/面板层级 | ✅ 有层级 / ⚠️ 层级不明 |

**ArkUI 提示**: 使用 `$r('sys.color.*')` 系统资源色，深色模式自动适配；避免硬编码 `#FFFFFF`。

### 5.4 启动页

> 来源: https://developer.huawei.com/consumer/cn/doc/design-guides/system-capabilities-launch-page-0000002311335748

| 验收项 | 通过标准 | 判定 |
|--------|---------|------|
| 内容简洁 | 仅展示 Logo/品牌等必要元素 | ✅ 简洁 / ❌ 元素过多 |
| 静态设计 | 以静态为主，动画最少化 | ✅ 静态 / ⚠️ 动画过多 |
| 视觉衔接 | 与应用首页视觉风格连贯 | ✅ 连贯 / ❌ 风格割裂 |
| 无广告 | 启动页禁止广告/促销信息 | ✅ 无 / ❌ 有广告 |
| 时长控制 | 0.3 ~ 0.8s 最佳 | ✅ 在范围 / ⚠️ 过长 / ❌ > 2s |
| 多设备适配 | 使用系统布局能力适配各设备 | ✅ 已适配 / ❌ 仅单端 |
| 结构类型 | 纯图标型 或 上下排版型 | ✅ 符合 / ❌ 非标准结构 |

| 系统手势冲突 | 不与系统边缘返回等手势冲突 | ✅ 无冲突 / ❌ 有冲突 |

### 7.2 光标交互
