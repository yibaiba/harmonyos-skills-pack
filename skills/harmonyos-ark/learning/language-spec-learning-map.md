# 语言规范学习地图（空项目可直接用）

## 学习目标
- 快速建立 HarmonyOS Ark 应用开发的语言与框架认知
- 用最短路径从“会查文档”到“能写基础应用”

## 推荐学习顺序
1. ArkTS 语言基础
- 入口: https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-get-started
- 目标: 理解语法特性、类型系统、装饰器与模块组织

2. ArkUI 声明式 UI
- 入口: https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkui-overview
- 目标: 理解组件结构、状态管理、布局与交互

3. Ability 与应用模型
- 入口: https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/abilitykit-overview
- 目标: 理解应用结构、生命周期与页面组织

4. 开发导读与快速入门
- 入口: https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/application-dev-guide
- 入口: https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/start-overview
- 目标: 串联从创建项目到运行调试的主路径

5. API 参考与实践
- 入口: https://developer.huawei.com/consumer/cn/doc/harmonyos-references/ability-arkts
- 入口: https://developer.huawei.com/consumer/cn/samples/
- 入口: https://developer.huawei.com/consumer/cn/codelabsPortal/serviceTypes/43
- 目标: 从概念学习进入可运行样例

## 每天 30 分钟学习法
1. 10 分钟读官方概念
2. 10 分钟抄写最小示例
3. 10 分钟改动示例并验证

## 学习完成标志
1. 能独立解释 ArkTS 与通用 TypeScript 的关键差异
2. 能写出一个有状态页面并完成页面跳转
3. 能定位并阅读目标 API 的官方参考页
