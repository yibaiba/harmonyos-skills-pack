# 风险等级自动判定规则（通用）

## 1. 判定输入

- 功能类型
- 数据敏感度
- 权限复杂度
- 第三方依赖
- 发布影响范围

## 2. 快速判定流程图

```
  功能涉及支付/账号/敏感数据/医疗/未成年人？
   ├── Yes ──▶ 🔴 High
   └── No
        涉及定位/推送/设备能力/第三方SDK/UGC？
         ├── Yes ──▶ 🟡 Medium
         └── No
              涉及跨端同步/后台任务/媒体上传？
               ├── Yes ──▶ 🟡 Medium
               └── No  ──▶ 🟢 Low
```

## 3. 触发词与等级

### High（任一命中即可）

| 类别 | 触发词 | 示例场景 |
|-----|-------|---------|
| 支付金融 | 支付、交易、提现、金融、钱包、订单、退款 | 应用内购买、虚拟货币充值、会员订阅 |
| 账号身份 | 账号体系、实名、身份认证、SSO、OAuth | 手机号注册、人脸识别登录、第三方登录 |
| 敏感人群 | 医疗健康、未成年人、教育收费、老年人 | 健康数据记录、学生付费课程、儿童模式 |
| 敏感数据 | 通讯录、精确位置、生物特征、相册全量 | GPS 轨迹记录、指纹/面容存储、通讯录上传 |
| 内容审核 | 大规模 UGC、用户评论、社区发帖、直播 | 论坛发帖、图文分享、弹幕/聊天 |

### Medium（未命中 High，命中任一）

| 类别 | 触发词 | 示例场景 |
|-----|-------|---------|
| 设备能力 | 定位、推送、相机、麦克风、蓝牙、NFC | 门店定位、消息推送、扫码功能 |
| 后台行为 | 后台任务、定时同步、Keep-alive、下载管理 | 后台数据同步、定时提醒、大文件下载 |
| 第三方集成 | 第三方 SDK、广告 SDK、埋点 SDK、地图 SDK | 广告展示、崩溃上报、第三方地图 |
| 跨端同步 | 跨端状态同步、多设备登录、云端同步 | 手机-平板数据同步、多设备在线 |
| 媒体处理 | 媒体上传、视频处理、图片编辑、文件导出 | 头像上传、视频压缩、PDF 导出 |

### Low（仅基础能力）

- 离线工具、静态展示、本地计算为主
- 无支付、无账号、无敏感数据
- 纯展示类页面、本地计算器、离线笔记

## 4. 评分辅助判定（可选）

- High 条件命中数 ≥ 1 → High
- 否则 Medium 条件命中数 ≥ 1 → Medium
- 否则 → Low

## 5. 第三方依赖风险

| 依赖类型 | 风险评估 | 验证要求 |
|---------|---------|---------|
| 广告/营销 SDK | Medium+ | 确认数据采集范围、隐私合规声明 |
| 支付 SDK | High | 沙箱环境全链路测试、证书有效性检查 |
| 地图/位置 SDK | Medium | 确认定位精度声明、后台定位行为 |
| 数据统计/崩溃上报 SDK | Low-Medium | 确认数据上传内容、合规声明 |
| 开源组件（无网络） | Low | 确认许可证兼容性、已知漏洞扫描 |

## 6. 数据驻留与合规考量

- 用户数据是否跨境传输？跨境需额外合规评审
- 是否涉及 GDPR / 个保法 / CCPA 管辖用户？
- 数据存储是否有加密要求？静态加密 + 传输加密
- 数据保留策略是否明确？用户删除账号后数据是否清除

> 涉及跨境数据传输或多法域合规时，风险等级至少为 Medium。

## 7. 对应验证动作与时间投入

| 风险等级 | 验证活动 | 预计时间 |
|---------|---------|---------|
| **High** | 完整回归（全链路 + 全异常） | 4-8 小时 |
| | 合规材料复核（隐私政策、权限声明） | 1-2 小时 |
| | 审核演练（用审核账号模拟审核流程） | 1-2 小时 |
| | 跨端与深色全量检查 | 2-4 小时 |
| | 安全基础扫描（密钥泄露、HTTPS） | 1 小时 |
| **Medium** | 核心链路回归 | 2-4 小时 |
| | 权限与异常路径回归 | 1-2 小时 |
| | 三端核心页面检查 | 1-2 小时 |
| **Low** | 基础功能与稳定性检查 | 1-2 小时 |
| | 深色模式关键页检查 | 30 分钟 |
| | 素材与文案一致性检查 | 30 分钟 |

## 8. 风险升级场景

### 自动升级规则
- 若发现生产级阻断缺陷 → 等级上调一级
- 若涉及新增敏感权限 → 最低 Medium
- 若新增支付或实名认证 → 直接 High

### 事件触发升级
- 线上崩溃率突增（> 1%）→ 等级上调一级
- 收到用户隐私投诉 → 直接 High
- 第三方 SDK 爆出安全漏洞 → 根据影响范围提升
- 政策/法规变更影响现有功能 → 重新评估等级
- 审核被拒且理由涉及合规 → 直接 High
