# 深色模式与多端适配清单（通用）

## A. 深色模式 — 基础

1. 支持系统主题跟随或显式主题切换
2. 主题切换后无需重启，实时生效
3. 核心页面文本与背景对比度 ≥ 4.5:1（WCAG AA）
4. 大号文本（≥ 18pt / 14pt bold）对比度 ≥ 3:1
5. 按钮、输入框、提示信息在深色下可辨识
6. 弹窗、Toast、对话框、Snackbar 在深色下不失真
7. 占位符文本在深色背景下可区分于正文

## B. 深色模式 — 进阶

> 以下项在 Medium/High 风险项目中为必查项，Low 风险可选查。

1. 图片处理：纯白/高亮图片添加暗色叠层或圆角容器
2. 图标/插画：单色图标跟随主题色；彩色插画验证对比度
3. 阴影与层级：深色下 elevation shadow 使用更浅表面色而非加深阴影
4. 状态栏与导航栏：前景色与系统栏背景匹配，无白色闪屏
5. 启动屏/Splash：深色主题下启动屏背景与首页衔接，无白色闪帧
6. 地图/图表/WebView：第三方视图内嵌内容支持暗色或有容器隔离
7. 分割线/边框：使用半透明白色（如 `rgba(255,255,255,0.12)`），非纯灰
8. 禁用态控件：在深色下仍可辨识为"不可操作"而非"消失"

## C. 多端适配 — 布局

1. 断点策略明确：小屏（< 600dp）/ 中屏（600-840dp）/ 大屏（> 840dp）
2. 手机、平板、桌面核心流程均可完成
3. 无内容截断、重叠、不可点击区域
4. 横屏模式下布局不破裂，关键按钮可达
5. 安全区域适配：刘海屏/挖孔屏/圆角屏内容不被遮挡
6. 折叠屏：展开/折叠切换后布局自动适应，无信息丢失

## D. 多端适配 — 交互

1. 输入方式兼容触控与键鼠（如适用）
2. 触控热区 ≥ 48dp × 48dp，避免误触
3. 键盘导航：Tab 顺序合理，焦点可见
4. 鼠标悬停态：桌面端关键控件具备 hover 反馈
5. 导航模式适配：手机用底部 Tab / 平板用侧边栏 / 桌面用顶部导航（或响应式合并）
6. 文字缩放：系统字号放大至 200% 后布局不溢出、不截断

## E. 一致性

1. 同一业务任务在多端路径一致
2. 术语、状态、按钮行为保持一致
3. 权限与异常提示在多端一致
4. 品牌色/主题色在深浅模式下均满足对比度要求
5. 动画/过渡在不同端保持一致体验或合理降级

## F. 快速冒烟测试脚本

> 按以下步骤依次执行，预计 10-15 分钟完成一轮。

1. **切换到深色模式**：系统设置 → 深色主题
2. **冷启动**：观察 Splash → 首页过渡，有无白色闪帧
3. **首页浏览**：检查列表/卡片/图片/图标可读性
4. **核心链路 A**：走完一条完整链路（如下单/提交/搜索）
5. **弹窗验证**：触发至少 1 个弹窗/Toast/底部弹层
6. **切回浅色模式**：验证实时切换无崩溃、无残留暗色块
7. **横屏测试**：旋转设备，检查首页 + 详情页布局
8. **平板/桌面端**（如适用）：重复步骤 2-6
9. **截图留档**：关键页面截图保存，便于回归对比

## G. 验证工具推荐

- **对比度检测**：Chrome DevTools → Rendering → Emulate CSS media `prefers-color-scheme: dark`
- **移动端检测**：系统设置切换深色 + 屏幕录制回放检查闪帧
- **响应式预览**：Chrome DevTools Device Toolbar / Firefox Responsive Design Mode
- **无障碍扫描**：Lighthouse Accessibility audit / axe DevTools 插件
- **截图对比**：逐页截图后用 diff 工具（如 BackstopJS / reg-suit）对比前后版本

## H. 常见失败模式

| 失败模式 | 表现描述 |
|---------|---------|
| 白色闪帧 | 深色模式启动时出现 0.3s 白屏，然后切换到暗色背景 |
| 幽灵文本 | 深色背景 + 深灰文本，肉眼几乎不可见 |
| 反色图标 | 浅色图标在浅色模式正常，深色下与背景融为一体 |
| 硬编码颜色 | 部分组件使用固定 `#FFFFFF` 背景，在深色下形成白色方块 |
| 平板拉伸 | 手机布局直接拉伸到平板，两侧大量空白或元素过大 |
| 键盘遮挡 | 输入框获焦后被软键盘遮挡，用户看不到输入内容 |
| 安全区溢出 | 底部按钮被系统手势条/导航栏遮挡，无法点击 |
| 字号溢出 | 系统字号放大后文字溢出容器，相互叠压 |
| 第三方白块 | WebView/地图/图表组件未适配暗色，形成白色矩形区域 |
| 导航死区 | 平板侧边栏展开后遮挡主内容，且无法关闭 |

## I. 适用范围说明

- 本清单适用于任意带 UI 的项目（App / Web / Desktop / Mini Program）
- 纯后端 / CLI 项目可跳过本清单
- 若项目不支持深色模式，A-B 节标记 N/A，仅执行 C-F 节
