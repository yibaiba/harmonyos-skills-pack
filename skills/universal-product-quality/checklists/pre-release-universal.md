# 发布前通用检查清单

> 每项标注严重级别：🔴 Blocker / 🟡 Warning / 🟢 Nice-to-have

## A. 功能完整性

1. 🔴 至少 3 条完整业务链路可独立走通
2. 🔴 每条链路存在明确成功/失败反馈
3. 🔴 每条链路包含异常处理（弱网/拒权/空态/服务异常）
4. 🟡 无占位符文本（lorem ipsum、TODO、测试文案）残留
5. 🟡 所有链接/跳转均可正确到达目标，无 404 或空白页
6. 🟢 非核心链路至少有 stub 或"即将上线"提示，而非无响应

## B. 体验一致性

1. 🔴 页面状态完整（加载、空态、错误、完成）
2. 🔴 操作路径可预测，关键入口可发现
3. 🔴 文案、截图、视频与实际功能一致
4. 🟡 返回/关闭操作行为符合预期，无导航死循环
5. 🟡 表单提交后有明确反馈（成功提示 / 跳转 / 列表刷新）
6. 🟢 动画/过渡流畅，无明显卡顿或跳帧

## C. 稳定性与可维护性

1. 🔴 核心链路无崩溃、无阻断
2. 🟡 日志与错误提示可用于定位问题
3. 🟡 配置与环境切换后行为一致
4. 🟡 连续快速操作（多次点击提交）不产生重复请求或崩溃
5. 🟢 后台切换/内存回收后返回，状态可恢复

## D. 发布可验证性

1. 🔴 使用全新安装包执行回归
2. 🔴 使用审核/测试账号复现关键流程
3. 🟡 发布前 24 小时重新检查活动/规则时效性
4. 🟡 版本号/构建号已正确更新

## E. 性能检查

1. 🟡 首屏加载时间 ≤ 3 秒（中等网络条件）
2. 🟡 页面切换/交互响应 ≤ 300ms
3. 🟡 列表滚动/动画帧率 ≥ 55fps，无明显掉帧
4. 🟢 内存使用无持续增长（操作 5 分钟后无明显泄漏）
5. 🟢 长时间使用不导致设备明显发热或电量骤降

### 性能目标（按风险等级）

| 指标 | Low | Medium | High |
|------|-----|--------|------|
| 首屏加载 | ≤ 4s | ≤ 3s | ≤ 2.5s |
| 页面切换响应 | ≤ 500ms | ≤ 300ms | ≤ 200ms |
| 列表滚动帧率 | ≥ 45fps | ≥ 55fps | ≥ 58fps |
| 内存增长（5 min） | < 30MB | < 15MB | < 10MB |

> Low 风险可放宽标准；High 风险按最严格标准执行。

## F. 无障碍检查

1. 🟡 文本与背景对比度 ≥ 4.5:1（WCAG AA）— [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)
2. 🟡 可交互元素具备 accessibility label — [axe DevTools](https://www.deque.com/axe/devtools/)
3. 🟢 关键流程可通过键盘（Tab / Enter / Esc）完成
4. 🟢 文字缩放至 200% 后布局不溢出
5. 🟢 屏幕阅读器可正确朗读页面核心内容 — [Lighthouse Accessibility](https://developer.chrome.com/docs/lighthouse/accessibility/)

## G. 安全基础检查

1. 🔴 代码/配置中无硬编码密钥、API Key、密码
2. 🔴 所有网络请求使用 HTTPS
3. 🔴 用户输入经过校验/转义，防止注入攻击
4. 🟡 认证 Token 存储安全（非明文、有过期机制）
5. 🟡 敏感操作（删除/支付/修改密码）有二次确认
6. 🟢 错误信息不泄露内部实现细节（堆栈、SQL、路径）
